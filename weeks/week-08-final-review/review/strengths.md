# Strengths

## Incident Debugging Mindset
Every week followed the same structured approach: observe symptoms,
check system state, read logs, identify root cause, apply fix, verify,
document prevention. This mirrors real production incident response.

## Container Troubleshooting
Hands-on experience diagnosing Docker runtime failures, OOM kills,
exit codes, image issues, and compose misconfigurations across
multiple weeks of incidents.

## CI/CD Troubleshooting
Diagnosed and fixed GitHub Actions pipeline failures including Java
version mismatches, invalid dependency versions, missing secrets,
wrong file paths, and Docker push failures.

## Root Cause Analysis
Every incident produced a documented root cause, not just a fix.
Understanding why something broke — not just how to fix it —
is the difference between reactive and proactive operations.

## Production-Style Documentation
Each week produced incident reports in a consistent format:
summary, investigation, root cause, resolution, prevention, timeline,
and evidence. This is the format used in real SRE and DevOps teams.

## End-to-End Delivery Pipeline
Built and debugged a complete delivery pipeline: code → CI/CD →
Docker image → registry → runtime → reverse proxy → user.
Every component in this chain was touched and broken at least once.
